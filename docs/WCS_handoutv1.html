Project Purpose

The WCS website provides a modern, mobile-first hub for families, coaches, and club leadership. It centralizes schedules, team news, coach resources, and supports future growth opportunities (shop, sponsors, and community programs).

Key Public Features

User-focused & mobile-ready
1
Home

Hero banner, Fan Zone links, Recent News and upcoming features — fast first impression for families and sponsors.

2
About

Club history, mission & values — ideal for building trust with new families and potential sponsors.

3
Teams & Team Pages

Dynamic team listings and dedicated team pages with coach info, news and schedules — a one-stop place for parents.

4
Schedules

Mobile-first calendar with color-coded events (games/practices) and quick filters for parents and coaches.

5
Practice Drills Library

Curated drills for coaches — supports volunteer and new coaches with structured practice plans.

Coaches Dashboard

  • Secure coach login to manage schedules, post team updates, and organize drills.
  • Instant sync to public pages so parents always see the latest information.
  • Designed to reduce administrative overhead and minimize communication errors.

Technical Snapshot

  • Frontend: Next.js (PWA), React + TypeScript, Tailwind CSS
  • Hosting: Vercel (production deployments, CDN)
  • Database / Auth: Supabase (Postgres + Auth + Storage)
  • Monitoring & Tools: Sentry, Vercel Analytics, Upstash Redis (rate-limits)

Security & Privacy

  • Supabase Auth with role-based access control (admin, coach, parent, guest).
  • Row-Level Security (RLS) for fine-grained data access.
  • All data encrypted at rest; CSRF protection on forms.
  • Privacy policy page included; GDPR checklist recommended for full compliance.

Quick Demo Checklist

  • Open homepage (desktop + phone) — show hero & fan zone
  • Teams → pick team → show coach, updates, schedule
  • Open Schedules → filter by team → open event details
  • Login as coach → add practice → show it appears
  • Show Drills library → filter → open drill detail

PWA vs Native App

Progressive Web App (PWA)

Installs from the browser to a phone home screen, works offline for basic views, updates instantly with deployments, and costs far less to build and maintain than native apps.

Native Mobile App

Built separately for iOS and Android, distributed via App Stores, offers deeper device integration but requires higher development & maintenance costs.

Recommendation: Start with the PWA (current approach) — add a native app only if later analytics justify the investment.
